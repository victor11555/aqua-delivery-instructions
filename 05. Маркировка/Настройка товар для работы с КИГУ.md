# Настройка товар для работы с КИГУ

## Кратко

Перед началом настройки обратите внимание на следующие моменты:


## Пошагово

1. GTIN у КМ и КИГУ разные – у КМ и КИГУ разные коды.
2. Если редактируем созданный товар с настроенным базовым товаром, то базовый товар отключаем.
3. Особенности работы после настройки – если ранее продажа мелкотарки велась без КИГУ, в первое время будут встречаться два типа заказов:
  - Требующие сканирования одной маркировки (для КИГУ);
  - Требующие сканирования каждой единицы товара (если КИГУ не применялся).
4. Настройка в 1С – перейдите:
1С → "Настройки" → "Еще больше возможностей" → "Общее"
и включите работу с несколькими единицами измерений.

# Настройки

1. Переходим в кабинет администратора и создаем категорию "Штучные товары" и отключаем отображение этой категории (Таким образом, если в категории с отключенным отображение включить отображение товаров, то товары будут видны только операторам)
![](image)

2. В созданной категории создаем штучный товар (Товар, который будет содержаться в упаковке). Товар рекомендуем заполнить следующим образом (Остальные настройки опциональны):
  1. Название - из названия должно быть понятно, что это штучный товар. Например: Природная питьевая вода газированная "AQUA" (АКВА), 0.5л;
  2. Отображение - включить;
  3. Включить маркировку - включить;
  4. GTIN - прописываем GTIN штучного товара;
  5. Цена - прописываем цену, как за штуку.
![](image)

3. Создаем товар упаковки (В любой подходящей под нее категории). Рекомендуем заполнить следующие поля (Остальные настройки опциональны):
  1. Название - из названия должно быть понятно, что это упаковка. Например: Природная питьевая вода газированная "AQUA", 0.5л, уп. 12 штук;
  2. Отображение - включить;
  3. Включить маркировку - включить;
  4. GTIN - прописываем GTIN КИГУ;
  5. Цена - прописываем цену, как за упаковку.
![](image)

4. Переходим в 1С - "Продажи" - "Номенклатура" и создаем штучный товар (Обратите внимание, что учет любого товара ведется в штука. Не нужно заводить отдельные товары упаковок). В разделе "Основное" заполняем следующие поля:
  1. Наименование - указываем название штучного товара, которое будет отображаться в программе. Например: Природная питьевая вода газированная "AQUA" (АКВА), 0.5л;
  2. Наименование для печати - наименование для печати может отличаться от наименования;
  3. Продукция ИС - Упакованная вода (Или другая продукция ИС, если пользуетесь)
![](image)

5. Переходим в раздел "Ед. изм" и нажимаем на "Создать". В открывшемся окне нужно заполнить следующие поля:
  1. Наименование - Нужно указать такое наименование, чтобы было понятно что это за упаковка. Например: Упаковка вода Aqua газированная 0.5;
  2. ОКЕИ - упак;
  3. Коэфициент - указываем сколько в упаковке содержиться штук товара.
![](image)

6. Переходим в раздел штрих коды и добавляем 2 штрих кода. Первый, для штучного товара, второй для КИГУ (В настройках штрихкода КИГУ нужно указать созданную единицу измерения).
![](image)

7. Переходим в раздел "CRM" - "Интернет-магазин" - "Обслуживание" - "Инструменты администратора" - "Соответствие объектов". Добавляем сопоставление товаров:
  1. Сопоставляем штучный товар
    1. Указываем ключ идентификации (Оставляем значение по умолчанию);
    2. Идентификатор магазина и идентификатор базы - указываем ID штучного товара из кабинета настроек
![](image)

    3. Ссылка на объект - выбираем номенклатуру в 1С
    4. Объект выгружен - проставляем признак
    5. Тип объекта - Справочник.Номенклатура
![](image)

  2. Сопоставляем товар упаковки
    1. Указываем ключ идентификации (Оставляем значение по умолчанию);
    2. Идентификатор магазина и идентификатор базы - указываем ID упаковки из кабинета настроек
![](image)

    3. Ссылка на объект - выбираем единицу измерения (Обратите внимание, что выбираем единицу измерения, а не номенклатуру)
![](image)

![](image)

    4. Объект выгружен - проставляем признак
    5. Тип объекта - Справочник.ЕдиницыИзмерения
![](image)

8. Готово.
