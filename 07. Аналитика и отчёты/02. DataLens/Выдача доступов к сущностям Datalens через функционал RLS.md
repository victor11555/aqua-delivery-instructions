# Выдача доступов к сущностям Datalens через функционал RLS

Далее будет описан процесс выдачи доступов к дашборду на примере единого воркбука для внутреннего использования. Все действия необходимо выполнять от админского аккаунта.

Воркбук для внутреннего использования: https://datalens.yandex.cloud/workbooks/p9ruj6zssohgc

Он используется в основном отделом аккаунт менеджеров. Также, он практически идентичен по набору данный с воркбуком для компаний-партнеров.

Воркбук для партнеров: https://datalens.ru/workbooks/mqgk698mlsjc8

Отличие их в основном только по навигации (фильтрам), т.к. у аккаунтов доступы ко множеству монолитов, а не к одному, как у компаний-партнеров. Суть выдачи доступов идентична для всех воркбуков.

1. Приглашение пользователя в организацию
![](image)

Организация - это пространство для совместной работы над аналитическими проектами, где пользователи могут создавать, редактировать и просматривать дашборды, наборы данных и другие объекты. Организация позволяет управлять доступом пользователей к этим ресурсам, назначая им различные роли (например, администратор, редактор, зритель и т.д.).

Поэтому, для дальнейшей работы необходимо добавить пользователя в организацию. Выдача доступов в даталенс производится через организацию org-adminaqua-deliveryru. Для этого, от пользователя потребуется EMAIL от аккаунта ЯНДЕКСА, т.к. сервис у яндекса.

Чтобы выслать приглашение, нужно перейти из боковой панели навигации:

Настройки сервиса > Управление пользователями > Пригласить пользователей

В открывшемся окне добавляем адрес почты и нажимаем Отправить приглашение.

Готово, приглашение в организацию отправлено!

2. Принятие приглашения пользователем
Есть нюанс при принятии приглашения, а именно, его нужно принимать будучи авторизованным только под тем пользователем, которому было выслано приглашение - иначе, если в браузере, с которого принимается приглашение есть авторизация в нескольких аккаунтах, то приглашение может отработать некорректно, в последствии доступа к данным не будет. Лучше всего авторизоваться в режиме инкогнито.

Чтобы принять приглашение, необходимо перейти из письма на почте, нажав на соответствующую кнопку - Принять приглашение. Далее пользователя перекинет в сервис DL, в котором надо будет авторизоваться и окончательно принять приглашение в организацию.

Готово, приглашение принято!

3. Настройка ролей (прав) пользователя в организации
Для этого переходим:

Настройки сервиса > Управление пользователями > Редактировать пользователей

Здесь есть два раздела:

- Участники организации - те, кто уже принял приглашение
- Приглашения - активные приглашения, которые еще не приняты
Важно: чтобы назначить роли - приглашение должно быть обязательно принято.

В разделе Участники организации находим нужного пользователя, проваливаемся в него и делаем следующие шаги:

Справа сверху Еще > Назначить роли > Добавить роль > в поиске ищем роль datalens.visitor > выбираем ее > Сохранить

![](image)

Роль datalens.visitor - это возможность просматривать данные к выданным сущностям без возможности редактирования и выдачи прав.

Готово, роль назначена!

Чтобы отозвать роль необходимо перейти в Еще > Диагностику доступов и напротив соответствующего ресурса нажав на три точки отозвать роль.

![](image)

4. Настройка группы
Группа - это объединение пользователей для единого доступа к данным. Прописывая настройки RLS - мы используем как раз группы. Поэтому переходим:

Настройки сервиса > Управление пользователями > Редактировать пользователей > в левом меню Группы

Находим необходимую группу, проваливаемся в нее > заходим в Участники > Нажимаем Добавить участника > находим пользователя > Сохранить

![](image)

Если группы нет, то создаем ее:

Справа сверху нажимаем Создать группу > даем название группы (если это партнер, то домен 3-го уровня-partner) > добавляем в описание название партнера, id и город, чтобы можно было в случае чего найти группу через поиск

![](image)

Готово, группа настроена!

5. Настройка RLS доступов
Учет ведется тут https://docs.google.com/spreadsheets/d/1hD6tZEK-Gpy_qCiXuVe8x9NXRRd_hcgBxYTmHlAPDDc/edit?gid=715540833#gid=715540833

А сам доступ выдается следующим образом:

Переходим в воркбук > переходим во вкладку Датасеты > открываем первый датасет > во вкладке Поля в 5-м столбце находим поле partner_id > нажимаем на ключик > вставляем запись на новую строку в таком виде:

'id партнера': @group:название группы

Сохраняем, в т.ч. датасет. Эту цепочку действий проделываем по всем датасетам. Лучше всего использовать сортировку по полю с ключиком, т.к. в некоторых датасетах может использоваться настройка RLS по двум полям.

![](image)

Готово, RLS настроен!

6. Выдаем доступ группе на воркбук
В воркбуке в правом верхнем углу нажимаем на замок (Доступ) > в открывшемся окне нажимаем + Добавить пользователя > находим нужную группу > добавляем > сохраняем. Обратите внимание, доступ выдаем уровня "Ограниченный просмотр".

![](image)

![](image)

Готово, доступ выдан!

Выдача доступов для просмотра других воркбуков, в которых используется ClickHouse производится аналогичным способом.

Чтобы выдать группе доступ ко всем данным не обязательно прописывать для каждого partner_id группу пользователей, достаточно прописать так:

*: @group:<название>

Для того, чтоб пользователь мог просматривать дашборд с данными, достаточно скинуть ссылку на сам дашборд! Ссылку можно получить открыв сам дашборд и через три точки сверху нажать на Поделиться.

Подробнее про RLS: https://yandex.cloud/ru/docs/datalens/security/row-level-security
