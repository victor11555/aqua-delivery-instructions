# Релиз приложения в RuStore

Инструкция понадобится сотруднику, который получил задачу по релизу приложения партнера в RuStore, обычно ставится задача отделом саппорта/аккаунт-менеджеров с указанием названия партнера и папки с графикой

Для релиза потребуется доступ к корпоративному аккаунту Вконтакте, на который придет код авторизации в Rustore, рабочий макбук с доступами к репозиториям и настроенным Java для создания подписей и Expo

В первую очередь необходимо зайти в консоль разработчика в Rustore (после авторизации по вк)
https://console.rustore.ru

![](image)

В правой части приложения необходимо нажать на кнопку "Добавить приложение"

![](image)

После этого вы заполняете основные данные о новом приложении (Платформа не полное ТВ и Название) и попадает на пустую карточку приложения магазина

![](image)

Следующим шагом необходимо выгрузить репозиторий проекта, запросить команду вы можете у разработчика, который занимается поддержанием сторов и работой с репозиториями приложений
Ссылку на репозиторий можно взять с файла https://docs.google.com/spreadsheets/d/1T7uvVoQ89FqWu8bcJpXyuhfpJWjNXc5-8aspkkuv7pQ/edit?usp=sharing

На рабочем компьютере нужно выгрузить репозиторий в папку проекта
cd documents/aqua

Далее ввести команду, полученную из файла или от одного из разработчиков

![](image)

Перейти в выгруженный репозиторий

![](image)

Нам сразу же потребуется сделать выгрузку файла .aab (Android Artefact Bundle) из Expo, если недавно был релиз или пре-релиз партнера, то может сохраниться доступ к .aab файлу, в ином случае нужно будет запустить новую сборку файла

Рассмотрим два примера

а) Файл еще есть, в таком случае в Expo вы увидите следующее

![](image)

Здесь остается только скачать и перейти к следующему шагу

б) Время архивной сборки закончилось и нужно делать новую, в таком случае вы увидите следующую картину

![](image)

Нам понадобится в репозитории партнера запустить ручную сборку артефактного проекта

В терминале в корне проекта нужно узнать какой аккаунт Expo сейчас авторизован

![](image)

Если авторизованный аккаунт содержит приложения для релиза в Rustore, в таком случае можно пропустить следующий подшаг

б1) Если нет приложения, необходимо авторизоваться в каждом аккаунте и найти на каком содержится приложение
https://expo.dev/

После того как аккаунет найден, можно вернуться обратно в терминал vscode и ввести команды
eas account:logout

eas account:login

Ввести нужные данные аккаунта expo и запустить сборку

eas build --platform android

После этого картина будет как на шаге а (в разделе builds в Expo)

![](image)

Скачанный .aab файл можно использовать на странице в Rustore

![](image)

Далее необходимо загрузить подписи, для этого можно идти по рекомендации RuStore
https://www.rustore.ru/help/developers/publishing-and-verifying-apps/app-publication/new-version-app/upload-aab

Нужно будет скачать pepk.jar и положить его в корень скачанного репозитория

Поочередно выполнить команды

java -jar pepk.jar --keystore /Users/minim1/Documents/aqua/{uid}App/android/app/{uid}.keystore --alias {uid} --output /Users/minim1/Documents/aqua/{uid}App/pepk_out.zip --encryptionkey=00006625917d74520c9ece4584e55dbad71b86ab43aba8f3ef9116dda0e3b54c3b235eef576f1c3f69dd2219c6df80e7b5f2400f48b7d87fdfe9b25ca6dfcd51404e8c0a --include-cert

Команда выше лишь пример того, где должен находить файл .keystore в проекте и куда будет выгружен зип файл с подписью (рабочий стол)

encryptionkey у каждого свой, нужно взять пример кода из формы

![](image)

Далее в терминале нужно выгрузить сертификат ключа загрузки

keytool -exportcert -alias aquapilot -keystore /Users/chus/Documents/aqua/AquapilotApp/android/app/aquapilot.keystore -rfc -file /Users/chus/Desktop/aquapilot.pem

alias во второй комнаде - это alias из первой кмонады, он должен быть такой же - это название ключа загрузки

Итоговые файлы для дальнейшей настройки должны быть следующие

![](image)

1. .aab
2. .pem
3. .peek_out.zip

![](image)

Указываем "Чувствительные разрешения" как на скриншоте и пишем обоснования
SYSTEM_ALERT_WINDOW - для использования платежных систем. READ_EXTERNAL_STORAGE и WRITE_EXTERNAL_STORAGE для логирования ошибок RECORD_AUDIO - техническая необходимость для работы камеры CAMERA - для оплаты через qr. POST_NOTIFICATIONS - для работы пуш-уведомлений. ACCESS_FINE_LOCATION и ACCESS_COARSE_LOCATION - для быстрого создания адреса.
