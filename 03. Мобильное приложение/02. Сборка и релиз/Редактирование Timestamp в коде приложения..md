# Редактирование Timestamp в коде приложения.

Данная инструкция пригодится специалисту, который занимается сборкой приложения. Редактировать Timestamp нужно в тех случаях, если новая сборка приложения с поставленной галочкой на создание нового Timestamp не создает его, из-за чего версия сборки не меняется. Это не позволит отправить новую версию в сторы. Чаще всего редактировать Timestamp приходится при ошибке в первичной сборке (если уже было отправлено в сторы) или ребрендинге партнера.

В первую очередь необходимо выгрузить локально проект с приложением. (В случае, если на рабочем компьютере его сейчас нет. Команду выгрузки можно запросить у Влада Новосельцева или специалистов тех.поддержки, которые занимаются сборкой.
Полученную команду нужно ввести в терминал, предварительно находясь в папке aqua 
cd documents/aqua

Если же проект уже был локально на компьютере, в первую очередь нужно подтянуть все изменения, которые есть.

После этого нужно открыть в редакторе кода полученный проект при помощи команды code в терминале (например, code russkayavoda)

После этого у вас откроется редактор кода VS Code, в котором вам необходимо поменять Timestamp в папке по пути android/app/build.gradle

Сформировать Timestamp вручную можно на сайте https://www.unixtimestamp.com/

Так же необходимо провести изменения в файле Xcode, для этого нужно открыть на рабочем компьютере папку с проектом (вручную) и найти в папке ios файл с расширением .xcworkspace
В двух таргетах нужно открыть General и поменять поле Build на скопированный ранее Timestamp
Первый таргет

После этого xcode нужно полностью закрыть (правой кнопкой по xcode в панели приложений и завершить)

Возвращаемся в редактор кода, в терминале пишем команды для отправки новых изменений

git add .

git commit -m  "fix: timestamp"

git push --set-upstream origin main

Изменения отправятся в AppCenter и начнет собираться новая версия приложения с новым номером версии.
